12. **Что такое in-memory Database?**
In-memory Database - это специализированная база данных, которая в основном полагается на внутреннюю память для хранения данных. Это позволяет минимизировать время отклика, устраняя необходимость доступа к стандартным дискам¹³.

13. **Что такое хэш-таблица?**
Хэш-таблица - это структура данных, используемая для быстрой вставки, поиска и удаления пар ключ-значение. Она работает на основе концепции хеширования, где каждый ключ переводится хеш-функцией в отдельный индекс в массиве. Индекс служит местом хранения соответствующего значения⁸.

14. **Как обеспечивается персистентность данных в СУБД Redis?**
Персистентность данных в Redis может быть обеспечена одним из двух способов: файл Append-Only (AOF) поддерживает запись операций записи, что позволяет восстановить данные, используя запись для восстановления базы данных до момента сбоя⁶.

15. **Назначение команд СУБД Redis: set, get, del, getset.**
- `set`: Устанавливает значение ключа.
- `get`: Возвращает значение ключа.
- `del`: Удаляет ключ и его значение.
- `getset`: Атомарно устанавливает ключ в значение и возвращает старое значение, хранящееся по ключу¹¹¹⁴.

16. **Назначение команд СУБД Redis: incr, decr, incrBy, decrBy.**
- `incr`: Увеличивает значение ключа на 1.
- `decr`: Уменьшает значение ключа на 1.
- `incrBy`: Увеличивает значение ключа на указанное число.
- `decrBy`: Уменьшает значение ключа на указанное число²⁶²⁷.

17. **Назначение команд СУБД Redis: mset, mget.**
- `mset`: Устанавливает значения для нескольких ключей.
- `mget`: Возвращает значения для нескольких ключей³⁵.

18. **Назначение команд СУБД Redis: hset, hget.**
- `hset`: Устанавливает значение поля в хеше.
- `hget`: Возвращает значение поля в хеше³⁷³⁹.

19. **Назначение команд СУБД Redis: hmset, hmget.**
- `hmset`: Устанавливает значения для нескольких полей в хеше.
- `hmget`: Возвращает значения для нескольких полей в хеше[^30^]³¹.

20. **Назначение команды СУБД Redis exists.**
Команда `exists` проверяет, существует ли указанный ключ¹⁹²².

21. **Назначение команды СУБД Redis multi.**
Команда `multi` используется для начала транзакции¹⁸.

22. **Назначение метода sendCommand.**
Метод `sendCommand` используется для отправки произвольной команды Redis. Это может быть полезно, когда вам нужно использовать новые или менее распространенные команды, которые не имеют соответствующих методов в вашем клиенте Redis.
