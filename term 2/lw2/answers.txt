**16. Поясните понятие «ORM»**
ORM означает Object-Relational Mapping. Это техника, которая позволяет вам взаимодействовать с вашей базой данных, как с объектами JavaScript. ORM делает это, преобразуя данные из базы данных в объекты JavaScript и наоборот.

**17. Какой дополнительный пакет необходимо установить для работы sequelize c Microsoft SQL Server?**
Для работы Sequelize с Microsoft SQL Server вам потребуется дополнительный пакет `tedious`.

**18. Поясните понятие «модель»**
В контексте ORM, модель представляет собой представление таблицы базы данных в коде. Модель определяет структуру данных, а также поведение и взаимосвязи с другими моделями.

**23. Поясните, что такое «хуки»? Какие они бывают?**
Хуки (или жизненные циклы) - это функции, которые вызываются перед или после выполнения определенных операций в Sequelize. Например, `beforeCreate`, `afterCreate`, `beforeUpdate`, `afterUpdate` и так далее.

**24. Поясните назначение функции sequelize Модель.hasMany**
Функция `hasMany` в Sequelize используется для установления отношений "один ко многим". Это означает, что одна запись в таблице может быть связана с несколькими записями в другой таблице.

**25. Поясните понятие «raw query»**
Raw query - это запрос, написанный на чистом SQL, который можно выполнить в Sequelize. Это полезно, когда вам нужно выполнить сложный запрос, который не может быть легко выражен через API ORM.

**26. Поясните понятие «транзакция»**
Транзакция - это группа операций с базой данных, которые выполняются как единое целое. Если любая из операций в транзакции не удается, все изменения, сделанные в рамках этой транзакции, откатываются, и база данных остается в исходном состоянии. Если все операции успешно завершаются, изменения фиксируются в базе данных.
